In this step we increase the challenge by adding an intelligence slider! This increases the complexity since we can no longer represent our state as a single number - instead we must use an object.

Don't worry about the intelligence code being very similar to the strength code - we don't need to refactor that now, instead focus on just making the state into an object. So add another action creator for the intelligence, and another case to the reducer.

Remember the useful ES6 trick: To create a new object from an old one but with additional props, do this: let newobj = {...oldobj, newprop: newval};
